## Why

FileFlash-Bridge 需要提供一种不依赖互联网和接收端安装 App 的近场投递方式，解决电脑、平板或其他手机把文件和临时文本快速发送到 iPhone、Android 手机上的高摩擦问题。现在推进 V1.0，是为了先交付局域网内可用、双端一致、可安全控制的“浏览器到手机”闭环能力，为后续历史记录等增强功能打基础。

## What Changes

- 新建基于 React Native 的 FileFlash-Bridge V1.0 移动端方案，统一支持 iOS 与 Android。
- 在 App 内集成本地局域网投递服务，支持在公共 Wi-Fi 和手机热点两种网络模式下对外提供访问地址。
- 提供浏览器可访问的响应式投递页，使其他设备无需安装 App 即可向手机上传文件、上传文件夹并提交粘贴文本内容。
- 提供二维码和直连地址展示，降低局域网配对和访问门槛。
- 增加基础安全控制能力，包括 `key` 访问控制、连接控制和手动停止服务。
- 明确双端文件保存、文本接收、权限申请、前后台限制、原生依赖边界和真机验证要求。

## Capabilities

### New Capabilities
- `local-transfer-server`: 在 React Native App 内启动和管理局域网投递服务，覆盖公共 Wi-Fi、热点模式、地址展示、二维码引导，以及入站文件和文本内容接收通道。
- `web-transfer-portal`: 提供浏览器访问的响应式投递页面，支持其他设备向手机上传文件、上传文件夹并提交粘贴文本内容。
- `transfer-security-controls`: 提供 `key` 访问控制、连接限制、服务启停和局域网访问安全控制。

### Modified Capabilities
- 无

## Impact

- 移动端代码：React Native 应用框架、共享业务层、iOS 原生能力接入、Android 原生能力接入。
- Web 资产：内嵌或动态提供的局域网投递页面、文件上传交互、文本提交交互、二维码展示能力。
- 平台能力：文件系统写入、文本接收区、局域网访问权限、热点/网络状态探测、前后台服务生命周期管理。
- 依赖与系统：本地 HTTP 服务库、二维码生成库、文件存储库、文本接收与状态管理能力。

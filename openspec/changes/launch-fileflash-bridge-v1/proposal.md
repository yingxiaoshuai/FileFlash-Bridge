## Why

FileFlash-Bridge 需要提供一种不依赖互联网和接收端安装 App 的近场文件传输方式，解决 iPhone、Android 手机与电脑、平板之间临时互传文件的高摩擦问题。现在推进 V1.0，是为了先交付局域网内可用、双端一致、可安全控制的核心闭环能力，为后续预览、剪贴板同步和历史记录等增强功能打基础。

## What Changes

- 新建基于 React Native 的 FileFlash-Bridge V1.0 移动端方案，统一支持 iOS 与 Android。
- 在 App 内集成本地局域网文件传输服务，支持在公共 Wi-Fi 和手机热点两种网络模式下对外提供访问地址。
- 提供浏览器可访问的响应式传输页，使其他设备无需安装 App 即可浏览文件、下载文件，并上传文件到手机。
- 提供二维码和直连地址展示，降低局域网配对和访问门槛。
- 增加基础安全控制能力，包括访问密码、连接控制、可选 HTTPS 能力评估与手动停止服务。
- 明确双端文件系统访问、权限申请、前后台限制、原生依赖边界和真机验证要求。

## Capabilities

### New Capabilities
- `local-transfer-server`: 在 React Native App 内启动和管理局域网文件传输服务，覆盖公共 Wi-Fi、热点模式、地址展示与二维码引导。
- `web-transfer-portal`: 提供浏览器访问的响应式传输页面，支持手机文件浏览、下载，以及其他设备向手机上传文件并展示进度。
- `transfer-security-controls`: 提供访问密码、连接限制、服务启停和传输安全控制，约束局域网访问风险与用户隐私暴露。

### Modified Capabilities
- 无

## Impact

- 移动端代码：React Native 应用框架、共享业务层、iOS 原生能力接入、Android 原生能力接入。
- Web 资产：内嵌或动态提供的局域网访问页面、上传下载交互、二维码展示能力。
- 平台能力：文件系统访问、相册或文档权限、局域网访问权限、相机扫码权限、热点/网络状态探测。
- 依赖与系统：本地 HTTP/WebDAV 服务库、二维码生成库、文件选择与存储库、可能的 HTTPS/证书支持方案。

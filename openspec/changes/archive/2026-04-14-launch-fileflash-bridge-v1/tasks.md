## 6. Incoming Content Handling

- [x] 6.1 实现手机端文本接收区：以「项目 / 会话」组织（类似 AI 聊天），每个项目下按时间线展示该项目内的多次浏览器文本提交；须维护**当前活跃项目**，新提交落入该项目，历史项目完整保留直至用户删除；**会话与消息须持久化到本机**（进程重启后不丢失）
- [x] 6.2 实现**新建项目**以开始下一轮文本分享；在项目内对单次提交支持一键复制、单条删除；删除**整个会话/项目**前展示确认框，明确说明**删除将清除会话数据及关联文件**，并提示**如有需要请先进入会话保存文件**；复制到系统剪贴板须由用户显式触发（如点按复制），避免静默覆盖已有剪贴板
- [x] 6.3 实现入站结果反馈：说明文件默认保存在 **App 内会话**、**不会自动出现在系统下载/相册**；提供会话内**保存到…/导出**入口与失败原因展示
- [ ] 6.4 真机验证：浏览器上传文件、上传文件夹或提交文本内容后，手机端能正确接收并反馈结果；多轮文本提交落在**当前活跃项目**内，新建项目后提交落入新项目；**杀进程重开**后会话与文件仍在；删除会话前出现完整警告文案
- [ ] 6.5 真机验证：App 选定共享后，浏览器可完整下载；大文件走分块路径；人为制造分块失败时验证 **4** 次尝试后整文件失败提示

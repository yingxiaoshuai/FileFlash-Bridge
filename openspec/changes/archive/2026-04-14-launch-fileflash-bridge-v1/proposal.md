## Why

FileFlash-Bridge 需要提供一种不依赖互联网和对端安装 App 的近场交换方式：电脑、平板或其他手机可通过浏览器向手机**投递**文件与文本，也可在用户在手机 App 内**显式选择共享文件**后，通过同一门户**分块下载**取回文件。现在推进 V1.0，是为了先交付局域网内可用、双端一致、可安全控制的“浏览器 ↔ 手机”闭环能力，为后续历史记录等增强功能打基础。

## What Changes

- 新建基于 React Native 的 FileFlash-Bridge V1.0 移动端方案，统一支持 iOS 与 Android。
- 在 App 内集成本地局域网投递服务，支持在公共 Wi-Fi 和手机热点两种网络模式下对外提供访问地址。
- 提供浏览器可访问的响应式门户页，使其他设备无需安装 App 即可向手机上传文件、上传文件夹、提交粘贴文本内容，并在手机端已共享的前提下**下载**指定文件（大文件分块传输，单分块失败自动重试且**至多再重试 3 次**，用尽仍失败则整文件报错）。
- 提供二维码和直连地址展示，降低局域网配对和访问门槛。
- 增加基础安全控制能力，包括 `key` 访问控制、连接控制和手动停止服务。
- 明确双端**会话内持久化存储**（不自动导出到系统目录）、入站**小文件压缩 / 大文件跳过压缩**、删除会话前的**文件清除提醒**、文本接收、权限申请、前后台限制、原生依赖边界和真机验证要求。

## Capabilities

### New Capabilities
- `local-transfer-server`: 在 React Native App 内启动和管理局域网传输服务，覆盖公共 Wi-Fi、热点模式、地址展示、二维码引导，以及入站文件与文本接收（**App 内会话持久化、可选压缩策略**）、**用户选定文件的出站共享与分块下载接口**。
- `web-transfer-portal`: 提供浏览器访问的响应式门户页面，支持向手机上传文件、上传文件夹、提交粘贴文本内容，以及**展示可下载的共享文件列表、分块下载与失败重试（单分块至多 4 次尝试）**。
- `transfer-security-controls`: 提供 `key` 访问控制、连接限制、服务启停和局域网访问安全控制。

### Modified Capabilities
- 无

## Impact

- 移动端代码：React Native 应用框架、共享业务层、iOS 原生能力接入、Android 原生能力接入。
- Web 资产：内嵌或动态提供的局域网门户页面、文件上传与**分块下载**交互、文本提交交互、二维码展示能力。
- 平台能力：**会话内文件系统读写与持久化**、**显式导出**时的用户授权路径、共享文件读取、文本接收区、局域网访问权限、热点/网络状态探测、前后台服务生命周期管理。
- 依赖与系统：本地 HTTP 服务库、二维码生成库、文件存储库、文本接收与状态管理能力。

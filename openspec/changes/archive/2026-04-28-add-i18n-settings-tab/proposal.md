## Why

当前 FileFlash-Bridge 的移动端工作台仍然是单页结构，用户需要在同一个界面里同时理解传输操作、项目内容和后续配置入口；随着功能增加，缺少稳定的设置入口会让首页继续变重。与此同时，现有文案仍以硬编码为主，缺少统一的国际化边界，不利于后续扩展英文界面与持续维护。

现在需要把“核心传输工作区”和“应用设置”分层，同时补齐最基础的双语能力，让用户可以在首页专注于投递与查看，在设置页中切换语言，而且 iOS 与 Android 都保持一致体验。

## What Changes

- 新增移动端国际化能力，为 App 内的首页 / 设置页 / 相关公共 UI 建立统一的文案资源与语言切换边界，首版仅支持中文和英文。
- 浏览器端门户页面也要支持中文 / 英文，并且 MUST 自动跟随 App 当前语言偏好（无需新增网页端语言切换入口）。
- 新增底部 Tab 导航，提供“首页”和“设置”两个一级入口；首页继续承载当前工作台，设置页承载应用级偏好项。
- 在设置页新增语言设置项，使用语言图标作为视觉入口，点击后展示下拉菜单或等价浮层，允许在“中文”和“English”之间切换。
- 新增本地语言偏好持久化，用户手动选择语言后，下次打开 App 仍保持上次选择。
- 保持现有局域网传输、共享导入、项目查看、安全模式与浏览器入口逻辑不变；本次不新增服务端接口、不修改传输协议、不要求新增系统权限。

## Capabilities

### New Capabilities
- `app-language-preferences`: 定义移动端语言资源、语言切换入口与语言偏好持久化行为。
- `mobile-tab-navigation`: 定义移动端底部 Tab 壳层，以及首页 / 设置两个一级入口的导航行为。

### Modified Capabilities
- `workspace-visual-refresh`: 调整刷新后的工作台视觉系统，使首页与设置页在新增底部 Tab 壳层下仍保持统一的 Paper 组件语义与视觉风格。

## Impact

- 主要影响移动端共享 React Native 代码，预计涉及 `App.tsx`、工作台拆分、设置页 UI、公共文案组织方式，以及与本地偏好读取相关的状态层。
- 方案预计仍以 React Native 共享代码为主实现；底部 Tab 与设置项优先复用现有 `react-native-paper` 组件能力。国际化运行时允许引入纯 JS 依赖，但不需要新增 Native Module。
- iOS 与 Android 都应共享同一套导航结构和语言设置逻辑；平台差异主要体现在安全区、返回键和浮层菜单定位，不涉及新的权限申请路径。
- 本次变更不改变局域网传输服务与协议语义、安全访问控制与文件存储模型；但会让浏览器端门户文案随 App 语言偏好渲染。隐私影响仅限本地保存用户选择的语言偏好。

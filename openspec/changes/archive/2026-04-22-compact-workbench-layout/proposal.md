## Why

当前主工作台把“连接 / 共享 / 模式”三个短状态放进大尺寸卡片，又将“复制链接”“刷新地址”等低频操作独立做成醒目按钮，导致首屏大量高度被状态与操作占用，而真正高频的共享文件列表和当前项目内容被挤到折叠以下。随着项目历史侧边栏已经稳定，继续保留这种信息密度偏低的首页布局，会降低用户对服务状态、共享文件和当前任务的感知效率。

现在需要对移动端工作台做一次更聚焦的信息架构压缩：在不改变局域网传输、访问模式、安全校验和共享文件能力语义的前提下，减少非核心状态的视觉体积，把操作入口靠近其作用对象，并把更多首屏空间让给共享文件和当前项目内容。这项调整同时适用于 iOS 与 Android，且可以继续完全基于 React Native 共享代码实现，无需新增原生模块。

## What Changes

- 将工作台顶部的“连接 / 共享 / 模式”从大卡片改为紧凑型横向状态摘要，放在标题区域下方，以图标、标签和数值的轻量表达替代高占位卡片。
- 重构“服务”模块的信息层级：服务在线/离线状态直接整合进服务标题与启动/停止主按钮，避免“未启动”状态在多个位置重复出现。
- 将“复制链接”“刷新地址”等操作与地址展示合并，改为地址栏右侧的紧凑图标操作；在服务未启动时隐藏或禁用这些操作，避免无效入口抢占视觉焦点。
- 将服务地址区在未启动时压缩为单行状态，服务运行后再展开为可读地址与操作区，减少空白占位。
- 调整首页内容排序和高度分配，使共享文件列表在常见手机屏幕首屏中可直接看到 3 至 4 条项目，提升用户对待处理共享内容的可见性。
- 将网络诊断和访问模式相关提示改为更轻量但仍可发现的状态项，例如小图标提示、次级说明或可展开区域，避免长期占用整行横幅。
- 保持现有传输协议、地址刷新逻辑、安全模式能力、二维码生成、项目历史 Drawer、文件共享与导出行为不变；本次不引入新的服务能力、页面路由或原生依赖。

## Capabilities

### New Capabilities

无。

- `local-transfer-server`: 调整移动端工作台在服务状态摘要、地址操作编排、网络诊断提示和共享文件首屏可见性上的产品要求，明确首页应采用更紧凑的信息布局而不改变传输能力本身。

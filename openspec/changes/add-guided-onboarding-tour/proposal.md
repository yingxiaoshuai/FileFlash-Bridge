## Why

当前 FileFlash-Bridge 的核心链路需要用户自己理解“启动服务 -> 打开地址/扫码 -> 在浏览器上传或发文本 -> 回到 App 查看和导出”的操作顺序。随着移动端工作台在持续压缩首屏说明文字，首次使用者更容易因为不知道下一步做什么而停在当前页面，因此需要一种可跳过、可逐步推进的交互式新手引导来降低上手门槛，同时保持页面本身的简洁度。

## What Changes

- 在移动端工作台新增首次使用引导，用分步高亮/浮层的方式介绍服务启动、访问地址、共享文件、当前项目内容等关键区域。
- 引导必须支持用户主动跳过、中途关闭、返回上一步，并在完成或跳过后不再自动反复打扰。
- App 端必须提供“重新查看引导”的入口，便于用户在忘记流程后再次打开教程。
- 引导仅做说明：通过框选/高亮告诉用户“在哪里、做什么”，不在引导里提供任何真实业务操作能力，也不替用户自动执行任何动作。
- 引导完成状态只在本地持久化，不新增服务端接口、不改变局域网传输协议，也不改变现有安全模式、上传下载、共享文件和项目数据模型。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `local-transfer-server`: 为移动端工作台补充首次使用的分步新手引导、跳过/完成后的本地记忆，以及手动重新打开引导的产品要求。

## Impact

- 主要影响移动端共享 React Native 工作台界面与状态管理，包括 [App.tsx](d:\company\FileFlash-Bridge\App.tsx) 及相关通用 UI/本地持久化逻辑；iOS 与 Android 默认共用一套引导逻辑，仅在安全区、触控反馈和系统返回行为上存在平台差异。
- 方案可继续完全基于 React Native 共享代码实现，预计不需要新增第三方原生库或自定义 Native Module。
- 安全、隐私和兼容性影响主要是本地记录“是否已完成/跳过引导”的轻量状态；不新增额外权限申请，不采集新的网络传输数据，也不改变现有访问控制和局域网服务暴露方式。

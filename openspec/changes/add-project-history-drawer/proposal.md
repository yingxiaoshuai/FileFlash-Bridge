## Why

当前 App 在宽屏布局下会常驻展示左侧“项目历史”列表，但缺少一个统一的左上角入口来收起、展开和管理历史项目；当用户只是想快速切换项目时，界面会长期被历史列表占据，首页信息层级也不够稳定。现在补上一个可展开的历史项目入口，可以让用户更自然地在“主页总览”和“历史项目管理”之间切换，同时保留已有的项目创建、删除和激活能力。

该能力需要在 iOS 和 Android 上保持一致，且无需引入新的原生依赖。现有项目存储、激活项目切换与删除确认能力已经具备，适合在这一轮通过 React Native 共享 UI 完成交互升级，并明确“从历史中选中项目后回到主页”的产品行为。

## What Changes

- 将当前常驻左侧的“项目历史”区域调整为左上角可触发的历史项目入口，支持展开与收起。
- 在展开层中展示全部历史项目，包含当前激活项目的选中态、基础统计信息，以及切换入口。
- 在展开层中提供“新建项目”和“删除项目”操作，并复用现有删除确认提示，避免误删关联文件。
- 明确项目切换后的导航行为：当用户从历史列表选中某个项目时，系统切换该项目为当前活跃项目，关闭展开层，并返回主页总览态。
- 保持浏览器投递仍然写入当前活跃项目，不改变已有的数据持久化、删除语义和跨平台共享实现方式。

## Capabilities

### New Capabilities
- 无

### Modified Capabilities
- `local-transfer-server`: 调整手机端项目历史管理入口与项目切换后的主页回退行为，补充历史项目展开管理、创建、删除和切换的界面要求。

## Impact

- 受影响代码主要位于 React Native 共享界面层与页面状态管理，包括 [App.tsx](d:\company\FileFlash-Bridge\App.tsx) 和 [useAppModel.ts](d:\company\FileFlash-Bridge\src\app\useAppModel.ts)。
- 受影响能力为本机项目/会话管理体验，底层持久化网关与浏览器投递协议预计无需新增 API，只需复用现有 `createProject`、`setActiveProject`、`deleteProject` 能力。
- iOS 与 Android 均交付相同交互，不计划引入新的第三方原生库或自定义 Native Module。
